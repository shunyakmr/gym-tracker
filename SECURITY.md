# Security Policy

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report issues privately via GitHub security advisories:

1. Go to the repository `Security` tab.
2. Click `Report a vulnerability`.
3. Provide clear reproduction steps and impact details.

If GitHub security advisories are unavailable, open a regular issue with
minimal detail and request private follow-up.

## Response targets

- Initial acknowledgment: within 7 days.
- Status updates: at least every 14 days until resolved or triaged.
