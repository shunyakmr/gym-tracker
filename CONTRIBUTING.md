# Contributing

Thanks for contributing to Training Tracker.

## Development setup

1. Fork and clone the repository.
2. Create a branch from `main`.
3. Make your changes.
4. Open `index.html` in a browser (or serve locally) and test core flows:
   - Quick log save
   - Program JSON edit/save
   - Dashboard rendering

## Pull requests

1. Keep PRs focused and small.
2. Include a short description of:
   - what changed
   - why it changed
   - how you tested it
3. If UI changes were made, include screenshots or short screen recordings.

## Code style

- Use clear names and small functions.
- Avoid adding dependencies for simple static-site behavior.
- Keep changes compatible with mobile-first usage.

## Reporting issues

Use the issue templates for bugs and feature requests.
